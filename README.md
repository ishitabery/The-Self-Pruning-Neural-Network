# The-Self-Pruning-Neural-Network
During training, a custom L1 Sparsity Loss is added to the standard Cross-Entropy loss:Because the L1 derivative is constant, it applies continuous downward pressure on the gate_scores. Unless a connection is critical for classification accuracy, its gate will collapse to 0, effectively pruning the weight from the network. The hyperparameter $\lambda$ controls the trade-off between network sparsity and classification accuracy.🛠️ Installation & RequirementsEnsure you have Python 3.8+ installed. You will need the following libraries:Note: A CUDA-enabled GPU is highly recommended for faster training, though the script will automatically fallback to CPU if unavailable.🏃‍♂️ UsageClone the repository and run the main script. The code will automatically download the CIFAR-10 dataset into a local ./data folder and begin training.📊 Expected OutputsThe script runs three separate experiments testing different $\lambda$ values (1e-5, 1e-4, 1e-3) to demonstrate the sparsity-vs-accuracy trade-off.Upon completion, the script generates:Console Report: A formatted table detailing final Test Accuracy and Sparsity percentages for each $\lambda$, alongside a per-layer sparsity breakdown.training_curves.png: A 3-panel plot showing Test Accuracy, Sparsity Level, and Classification Loss over 30 epochs.gate_distributions.png: Histograms showing the final distribution of gate values. Successful pruning is visually confirmed by a massive spike at 0.0.📁 Repository Structure👨‍💻 Author
